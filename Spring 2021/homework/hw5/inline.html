In-Line Elements

In-Line elements are tags in HTML that typically take up 1 line. For example, this bold, strong text houses the <strong> and </strong> tags. This italicized text utilizes the <i> and </i> tags.

Here are a few In-Line tags used in HTML.

Term Opening Tag Closing Tag What It Does
strong <strong> </strong> Bolds Text
i <i> </i> Italicizes Text
h1 <h1> </h1> Acts as a Primary Header.
h2 <h2> </h2> Acts as a Secondary Header.
h3 <h3> </h3> Acts as a Tertiary Header.

Again, these tags usually last one line. Unless there is a lot of text inside, it is one line. Thus, it is called in-line. However, there are other things in HTML that are 'inline'. It is not just restricted to inline elements.

Connotation

In-Line also has different connotations. One is used in CSS, when defining where to put the CSS. One such style is In-Line CSS (the others are External CSS and Embedded CSS). This means adding more code inside block-level elements. Block elements, you ask?

I'm talking about the elements like <p> and </p> or <html> and </html>.
You know, the big boys of the tags (they can house in-line elements like <strong> and </strong>).

Here are a few examples.

  • <p style="text-align:center; color:black;">Hello</p>
  • <ul style="list-style-type: square;">Hello</ul>
  • <article style="text-align: center;">Hello</article>

Click Me to Learn More about In-Line and Block-Level Elements.
You may ask. All these 3 options to do CSS... Which is the best? Well, the answer goes to External CSS. In-Line CSS looks awkward and is difficult to understand for beginners. Embedded CSS is similar to External CSS, but is defined in the HTML file. However, if multiple pages are using the same CSS rules, you would have to copy and paste all that code into all the other HTML files. External CSS accomodates for all of these flaws that the other 2 have.
